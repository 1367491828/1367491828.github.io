mysql数据库的事务

Mysql数据的事务

事务定义：

事务：一个最小的不可再分的工作单元；通常一个事务对应一个完整的业务。一个完整的业务需要批量的DML(insert、update、delete)语句共同联合完成。事务只和DML语句有关，或者说DML语句才有事务。这个和业务逻辑有关，业务逻辑不同，DML语句的个数不同。

事务的四个特性（ACID）：

1.原子性(A)： 事务是最小单位，不可再分。

2.一致性(C)： 事务要求所有的DML语句操作的时候，必须保证同时成功或者同时失败。

3.隔离性(I)： 事务A和事务B之间具有隔离性。

4.持久性(D)：是事务的保证，事务终结的标志(内存的数据持久到硬盘文件中)。

相关术语 ：

  • 开启事务：Start Transaction
  • 事务结束：End Transaction
  • 提交事务：Commit Transaction
  • 回滚事务：Rollback Transaction

隔离性：

隔离的四个级别：

  • 读未提交：read uncommitted
  • 读已提交：read committed
  • 可重复读：repeatable read（Mysql默认级别）
  • 串行化：serializable

查看数据库隔离级别（Mysql5.7之后）：

show variables like 'transaction_isolation';或者select @@transaction_isolation;

1.read uncommitted

- 事务A和事务B，事务A未提交的数据，事物B可以读取到 - 这里读取到的数据叫做“脏数据” - 这种隔离级别最低，这种级别一般是在理论上存在，数据库隔离级别一般都高于该级别

2.read committed

- 事务A和事务B，事务A提交的数据，事务B才能读取到 - 这种隔离级别高于读未提交 - 换句话说，对方事务提交之后的数据，我当前事务才能读取到 - 这种级别可以避免“脏数据” - 这种隔离级别会导致“不可重复读取” 。

3.repeatable read

  • 事务A和事务B，事务A提交之后的数据，事务B读取不到
  • 事务B是可重复读取数据
  • 这种隔离级别高于读已提交
  • 换句话说，对方提交之后的数据，我还是读取不到
  • 这种隔离级别可以避免“不可重复读取”，达到可重复读取
  • 比如1点和2点读到数据是同一个
  • MySQL默认级别
  • 虽然可以达到可重复读取，但是会导致“幻像读”

4.serializable

- 事务A和事务B，事务A在操作数据库时，事务B只能排队等待 - 这种隔离级别很少使用，吞吐量太低，用户体验差 - 这种级别可以避免“幻像读”，每一次读取的都是数据库中真实存在数据，事务A与事务B串行，而不并发。

设置隔离级别

READ UNCOMMITTED
– READ COMMITTED
– REPEATABLE READ
– SERIALIZABLE
• 例如： SET TRANSACTION ISOLATION LEVEL REPEATABLE READ;

设置隔离作用范围

事务隔离级别的作用范围分为两种：
– 全局级：对所有的会话有效
– 会话级：只对当前的会话有效
• 例如，设置会话级隔离级别为READ COMMITTED ：
mysql> SET TRANSACTION ISOLATION LEVEL READ COMMITTED；
或：
mysql> SET SESSION TRANSACTION ISOLATION LEVEL READ COMMITTED；
• 设置全局级隔离级别为READ COMMITTED ：
mysql> SET GLOBAL TRANSACTION ISOLATION LEVEL READ COMMITTED；

mysql的数据类型

Mysql的主要数据类型

数字类型：

主要的数字数据类型：TINYINT, SMALLINT, INT , BIGINT, FLOAT , DOUBLE , DECIMA(M,N);

TINYINT: 占一个字节，带符号时范围为：-128127，无符号时为0255.

SMALLINT: 占两个字节，带符号时范围为：-2的15次方2的15次方-1（-3276832767），无符号时为02的16次方-1（065535）

INT : 占四个字节，带符号范围为-2的63次方2的63次方-1（-21.47亿21.47亿），无符号为（0~42.94亿）

BIGINT: 占八个字节带符号范围为：（-922.3万万亿-922.3万万亿），无符号为（01844.6万万亿）

FLOAT（M，N）: 占用四个字节，M表示显示长度，N表示小数位长度。当输入小数位长度大于N时，会进行四舍五入保留N位小数。

DOUBLE（M，N）: 占用八个字节，精度更大的小数和float使用方法类似。

DECIMA(M,N): 和float类似只是比float精度更高。

  • 定义无符号的数字，在类型后面空格 接 unsigned

文本类型：

主要的字符串类型数据：char , varchar, text

CHAR(n): 固定长度字符串 ，最大255。

VARCHAR（n）: 可变长度字符串，最大长度65535，varhar的最大长度是65535，但是有1-2个字节是用来记录数据大小的，所以有效的字节是65533。。

TEXT: 大文本，不支持全文搜索，不支持默认值。

编码不同，varchar(L)里的最大值L是不同的。
utf8中，一个字符占3个字节，所以L = 65533/3 = 21844，道理来说uft编码下L最大为21844
gbk编码中一个字符占2个字节，所以L = 65533 / 2 = 32766，道理来说gbk编码下L最大为32766

在Mysql中，汉字和字母都是一个字节,varchar(2)只能存两个字母或汉字理论最大值按上诉，21844或32766

总结：

  1. 如果数据长度都一样，就用char，比如身份证号、手机号
  2. 如果数据长度不一样，就用varchar，比如名字、地址，但要保证不越界
  3. 定长的磁盘空间比较浪费，但查询收索效率高
  4. 变长的磁盘空间比较节省，但查询收索效率低,

时间日期类型

主要的类型：DATE, DATETIME, TIMESTAMP

DATE: 时间日期格式 ‘yyyy-mm-dd HH:ii:ss’ 表示范围从1000到9999，占用八字节

DATETIME: 日期 ‘yyyy-mm-dd’，占用三字节

TIMESTAMP: 时间戳，从1970年开始的 yyyy-mm-dd HH:ii:ss格式和datetime完全一致，占用四字节

enum和set

enum：枚举，单选类型
语法：
enum(‘选项1’, ‘选项2’, ‘选项3’…)

set：集合，多选类型
语法：
set(‘选项1’, ‘选项2’, ‘选项3’…)
例：建一个兴趣爱好调查表votes，比如（游泳、跳舞、弹吉他、滑雪、冲浪）中选择（可以多选），（男、女）【单选】

如果我们想要查询所有喜欢跳舞的人，就需要使用集合查询使用find_ in_ set函数

mysql的使用

Mysql的基本使用

1.登录mysql数据库（通过cmd方法）：

windows系统下，进入mysql文件目录下的bin目录，文件栏输入cmd，回车，打开cmd面板.

mysql -u root -p //该语句通过root账号登录，可以改变账号，输入后，提示输入密码

2.查看数据库

show databases; //查看数据库中有那些数据库

3.切换（进入）数据库

use databaseName; //use后面的是数据库名称

4.查看包含数据库表

show tables; //查看数据库中所有表

5.查看表的结构

​ 1.desc tableName; //查看数据库表的结构描述

​ 2.show create table tableName; //查看生成表的ddl语句,可以看出表的结构，以及引擎等信息

6.创建数据库

create database databaseName; //创建名为databaseName的数据库

7.创建数据库表（先进入所在的数据库）

create table tableName(

id int(20) primary key auto_increament,

password varchar(255) not null,

updatetime datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

primary key(id,class_id) //设置多字段主键

foreign key(id) references class(id) 设置外键//

)

​ 设置字段限制

​ primary key //设置为主键

​ foreign key //设置外键

​ auto_increament //设置为自增

​ not null //不能为空

​ default ”value“ //设置字段默认值

​ unique //设置字段值唯一

​ DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP //设置值默认为当前时间

​ comment ”字段描述“ //在字段约束后添加字段描述

增加语句

​ insert into user(id,password,nickname) values(10000,'123456','码呀');

  • 如果表后面没有字段就需要将所有字段的值都放在value中*
  • 可以同时插入多条数据，在一条数据后面加’,’逗号，在加括号包含数据可以一条语句插入多条数据。

查询语句

全部查询：

​ select * from user;

投影查询：

​ select id , password from user; //查询id,和password字段

​ 设置字段别名

​ select id , password PASSWORD from user; //在查询字段后面空格然后接别名

​ select id , password as PASSWORD from user; //在查询字段后面as然后接别名

条件查询：

​ select password,email from user where id = 123456; //查询id=123456的数据password,email 字段

排序查询：

​ select * from score order by score DESC；

​ DESC为升序，ASC为降序

​ 有where 条件的order by语句跟在where语句后面

​ select * from score order where class = 1 by score ASC；

分页查询：

​ select mathscore from score where class = 1 order by DESC limit 10 offset 0; //从分页查询

​ offset = (pagenum - 1)*pagesize

​ OFFSET超过了查询的最大数量并不会报错，而是得到一个空的结果集

​ OFFSET是可选的，如果只写LIMIT 15，那么相当于LIMIT 15 OFFSET 0。

​ 在MySQL中，LIMIT 15 OFFSET 30还可以简写成LIMIT 30, 15。

​ 使用LIMIT OFFSET分页时，随着N越来越大，查询效率也会越来越低。

聚合查询：

​ SELECT COUNT(*) boys FROM students WHERE gender = 'M';

SUM 计算某一列的合计值，该列必须为数值类型
AVG 计算某一列的平均值，该列必须为数值类型
MAX 计算某一列的最大值
MIN 计算某一列的最小值
COUNT 计算列的行数

​ MAX()和MIN()函数并不限于数值类型。如果是字符类型，MAX()和MIN()会返回排序最后和排序最前的字符。

​ 如果聚合查询的WHERE条件没有匹配到任何行，COUNT()会返回0，而SUM()、AVG()、MAX()和MIN()会返回NULL。

​ 聚合查询的字段列中，只能放入分组的列，也可以使用多个列进行分组，查询出的结果按符合两个要求来分组。

多表查询：

SELECT
s.id sid,
s.name,
s.gender,
s.score,
c.id cid,
c.name cname
FROM students s, classes c
WHERE s.gender = 'M' AND c.id = 1;

​ 多表查询又称笛卡尔查询，使用笛卡尔查询时要非常小心，由于结果集是目标表的行数乘积，对两个各 自有100行记录的表进行笛卡尔查询将返回1万条记录，对两个各自有1万行记录的表进行笛卡尔查询将返回1亿条记录。

连接查询：

​ INNER JOIN只返回同时存在于两张表的行数据。

​ RIGHT OUTER JOIN返回右表都存在的行。如果某一行仅在右表存在，那么结果集就会以NULL填充剩下的字段。

​ LEFT OUTER JOIN则返回左表都存在的行。

​ FULL OUTER JOIN，它会把两张表的所有记录全部选择出来，并且，自动把对方不存在的列填充为NULL。

​ JOIN查询仍然可以使用WHERE条件和ORDER BY排序。

SELECT s.id, s.name, s.class_id, c.name class_name, s.gender, s.score
FROM students s
LEFT OUTER JOIN classes c
ON s.class_id = c.id;

修改语句：

​ UPDATE <表名> SET 字段1=值1, 字段2=值2, ... WHERE ...;

​ 如果WHERE条件没有匹配到任何记录，UPDATE语句不会报错，也不会有任何记录被更新。

​ 要特别小心的是，UPDATE语句可以没有WHERE条件，例如：

​ UPDATE students SET score=60;

​ 这时，整个表的所有记录都会被更新。所以，在执行UPDATE语句时要非常小心，最好先用SELECT语句来 测试WHERE条件是否筛选出了期望的记录集，然后再用UPDATE更新。

删除语句：

​ DELETE FROM <表名> WHERE ...;

​ 如果WHERE条件没有匹配到任何记录，DELETE语句不会报错，也不会有任何记录被删除。

​ 要特别小心的是，和UPDATE类似，不带WHERE条件的DELETE语句会删除整个表的数据。

定义模型：

定义：

sequelize默认给数据库添加id,createdAt,updatedAt三个字段，id为从1递增的数值，createdAt,updatedAt为datetime数据类型。

timestamps：boolean,当为true时，数据库中会默认设置,createdAt，updatedAt,两个字段，这两个字段为mysql的datetime类型，分别为创建的时间和上次该条数据修改的时间。

type: datatypes,约束字段的数据类型，具体类型如下面列出的数据。

allowNull:boolean，该字段是否允许为空。

defaultValue:anytype，该字段的值得类型。

set(value){ this.setDataValue(‘key’,value.toUpperCase())}: 首先获得要改变的值，再根据setDataValue方法赋值给某个字段（一般为自己),后接方法，方法可以在外部定义，但是方法必须返回加工后的值，change(value).

vuex的模块化使用

vuex

vuex的作用：

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。通俗来讲就是管理多个组件的共享数据，如果多个组件共享数据使用事件总线会很麻烦，这种数据共享方式比较麻烦，所以有了一个统一管理共享数据的工具。

1.下载vuex插件：npm install vuex --save

2.在项目src目录下新建store文件夹，并且在下面新建一个index.js文件，如果其他文件导入该store文件，该文件文件下只有这个index文件，则默认会找到这个index文件。

父子组件之间的传值

父组件传值给子组件

技术：属性绑定+数据拦截。

特点：数据具有响应式。

使用：

1.给子组件标签上使用 v-bind:attr="data"对子组件进行属性绑定，属性名称需为非标签属性，绑定之后正常情况下，是会在子组件的$attrs中可以看到该属性，并且在子组件的根标签上也可以看到到该属性。这就是属性绑定，因为使用的时v-bind的方法绑定，所以数据发生变动值都会传递下去，所以是具有响应式的。

2.在属性绑定后，子组件中根标签可以看到传递过来的属性，虽然可以在子组件中通过this.$attrs.attr来获取某个属性值，但是这样太过于麻烦，所以出现了props属性，在该属性下可以配置拦截的数据，如果拦截过的数据，就不会出现在子组件的根标签上，并且也不会出现在子组件的$attrs中，而出现在子组件实例的props属性中。

3.props的使用：

  1. props取值为数组时，如props:["name","age"]则会拦截子组件标签上传递的属性，拦截后这两个数据，name,和age就相当于数据仓库中的实例，可以在页面中使用花括号，计算属性等方法，所以该方法能从父组件传值到子组件。

  2. 当props取值为对象时：如

    props:{

    ​ name:{

    ​ type:String, //需要传递的值的类型，否则报错，但是还是会将传递的值显示在页面

    ​ default:”默认值”， //未传入值，或取值未undefined取默认值

    ​ required:boolean,//是否必须传值

    ​ validator:function //判断是否符合要求，如邮箱验证类似，如果符合则return true,

    ​ }

    }

    type:取值为数据类型如Number,String,Boolean,Object,Array等，不符合直接报错

    deafult:取值，更具类型来取默认值，其生效时只有在父数据仓库中将该值定义未undefined或没有绑定该属 性时，才会使用该默认值。

    required:当取值为true时，default大部分时候都没有用，除非传值的为undefined。

    validator: 取值为函数，当符合条件时返回true，不符合时则会报错，在这不能检测数据类型，return为false时为报错，但是页面还是会显示父组件传递的值，或默认值。

    取值为对象时仅对于开发人员有用，所以一般开发都是使用对象取值。

Tips:

在子组件上绑定的样式会往子组件根节点上传，不要数据拦截，标签属性不会被拦截。

子组件数据传递给父组件

技术：事件绑定+事件触发。

特点：数据不具有响应式。

使用：

1.在子组件中标签中添加事件，事件名需要自定义，并且不能使用驼峰命名，并且函数后面不能加括号，加括号表示函数不会传递参数，后面触发事件就不能传递参数了。同一个事件可以绑定多个函数，可以用数组方法绑定多个函数，触发事件后，都会执行。

2.在绑定事件后，子组件在生命周期的beforeCreate之前就会将该事件添加进去vue实例的私有属性_events中，__events中含有所有的自定义事件，而且每个事件对应一个数组，每个数组都只含有一个函数，这个函数就包含了触发事件的所有函数。

3.触发事件可以通过触发_events中的函数去触发事件，但是，_events属于私有属性，不太好，所以就有了$emit函数，使用$emit(“事件名”，参数)，如果事件名的是子组件的事件名，那么这个方法可以触发事件。

4.在父组件中定义触发的事件函数，函数中由子组件中的参数变成函数的实参，所以可以拿到子组件中传来的值，将该值取出赋值给父组件的数据仓库属性，这样就拿到了子组件传来的值。

5.该方法只能通事件的触发来实现值的传递，所以不是响应式。如果要响应式也可以通过改造，通过监控属性watch，监控传递的属性，如果有改变则触发$emit，并把参数传递出去，就实现了响应的功能效果。

Tips:

1.用于数据传递的子组件事件绑定，不要使用标签定义的已知事件，使用自定义事件，事件名不要使用驼峰方式。

2.项目开发中，存在需要将父组件的方法绑定到子组件根标签上，事件需要加.native修饰符，则在子组件根标签上还能看到事件，否则都会在_events中。

vue中$watch方法和watch监控属性的区别

watch属性监控器

watch:{attribute: function(newvalue,oldvalue){......}}

watch属性的作用是对数据仓库属性的监控。

对于数据为引用类型数据时（如对象，数组），如果单单对于该数据来监控，对于修改内部的属性，增添等修改将不会起到数据监控的作用，因为引用数据类型在栈中存的是在堆中存储的地址，所以，只要未改变地址就不会触发数据监控函数，只有对数据进行重新赋值，才会改变地址导致堆中的值发生改变。如果开启深度监视，deep:true时，会对引用数据类型内部进行监控，如果属性值发生改变，就会触发数据的监控函数，但是该方法是无差别监视，修改任意属性都会触发监控函数，所以称为无差别监控。而对于引用类型数据的内部属性，如果在初始化vue实例中就有的属性，vue会进行递归对内部的属性也进行添加数据劫持，所以数据内部的属性也是具有数据监控的作用，但是如果在后面新添加的属性就没有数据劫持。如果要只对某个属性进行监控 “Obj.attribute”:function(){}就能对数据进行监控。

​ 上述为对于对象而言，而对于数组来说有一些不同，其中数组有变异方法：有push,pop,shift,unshift,splice,sort,reverse,这些都是变异方法，且都是改变原数组的方法，能够触发引用数据类型的监控，效果也未发生改变。但是改变数组长度，通过数组下标来修改数组值得方式不能触发数组的监控函数。

$watch实例方法

$watch实例方法在实例外部使用，返回的是一个取消监控的函数，执行它就可让该属性失去监控效果，而且该方法可以监控多个数据仓库属性或其内部属性。

let unwatch = vm.$watch(exOrFn,callback,[options]);

unwatch();

第一个参数为表达式或者函数，表达式为某个数据仓库属性或其内部属性，函数时要return一个值，该作用用于关联监控多个属性， 可以为表达式也可以为对象，或数组，如{a:this.a,b:this.b}，它将监控a属性和b属性，且callback里的newvalue和oldvalue就是按这个格式来的，如果是数组则为数组形式，options参数为handler，和immediate,和deep。

两者区别：

两者都是对数据进行监控。

1.使用$watch可以取消某个属性的监控，而watch则只能在实例销毁后才能取消监控。

2.$watch可以同时监控多个属性，如一个相加例子，如果一个任何一个值改变，都需要重新计算，所以在watch中在两个属性中都需要重复定义一样的监控函数，而$watch可以监控多个属性，只要一个函数，所以有特殊作用。

vue中过滤器filter的使用

过滤器

作用：对数据进行包装处理

只有两种情况下使用：

1.页面中花括号内使用：//\{\{ data | filtername \}\} 或//\{\{ data | filtername() \}\},后者可以传递参数，但是第一个参数是管道符左侧的数据，不显示书写进去参数内。

2.v-bind:hj="data | filtername||filtername（）"，v-bind绑定数据时可以使用。

过滤器与计算属性的区别：

1.computed计算属性一般需要依赖一个数据仓库中的数据，其具有数据包装与数据仓库属性监控的作用，而过滤器只具有数据包装作用，其参数为管道符左侧的数据和传入的其他参数，而且具备数据仓库属性监控作用，因为响应的原因是因为花括号内管道符左侧变动，导致过滤器重新执行一遍。

2.computed计算属性具有缓存功能，它在同一个实例中，只要依赖的实例仓库属性没有发生改变，多次使用该计算属性的结果都是一样的，它使用的就是第一个的缓存结果，而过滤器使用每次使用都会执行一次过滤器定义的函数，所以在一定条件下效果相同但是效率是不一样的。

局部过滤器

在vue实例中定义局部过滤器：

filters : { key : value }

key：key的取值为字符串，其为过滤器的名称，所以其值要唯一。

value：取值为function ，执行数据包装操作，其必须要有返回值 return 返回值。

全局过滤器

定义在全局的过滤器，可以在不同的vue实例中使用。

定义：Vue.filter("key",function(){.....})

key同样也是过滤器名称。

function为包装数据处理，和局部过滤器的使用一致。

注意：

1.全局过滤器一定要在Vue实例定义之前就要定义，否则不生效。

2.全局过滤器和局部过滤器可以在同时出现，但是如果过滤器名称相同时，局部过滤器的优先级要更高，所以执行局部过滤器。
