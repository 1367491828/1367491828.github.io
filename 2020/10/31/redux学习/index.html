redux学习

redux的使用：

1.下载redux:

npm install redux --save

2.redux的组成：（ action, reducer, store）

action: 是把数据从应用传到 store 的有效载荷。(就是设置传递（修改）数据固定格式，修改数据只能按照这个格式修改)。

reducer: 指定了应用状态的变化如何响应 actions,并发送到 store 的，记住 actions 只是描述了有事情发生了这一事实，并没有描述应用如何更新 state。

store:

  • 维持应用的 state；

  • 提供 getState() 方法获取 state；

  • 提供 dispatch(action) 方法更新 state；

  • 通过 subscribe(listener) 注册监听器;

  • 通过 subscribe(listener) 返回的函数注销监听器。

三大原则：

单一数据源。

整个应用的 state 被储存在一棵 object tree 中，并且这个 object tree 只存在于唯一一个 store 中。

State 是只读的。

唯一改变 state 的方法就是触发 action，action 是一个用于描述已发生事件的普通对象。

使用纯函数来执行修改（reducer必须是纯函数）。

为了描述 action 如何改变 state tree ，你需要编写 reducers。

export const ADD_COUNTER = ‘add_counetr’
export const DEC_COUNTER = ‘dec_counetr’

reducers.js

import { ADD_COUNTER, DEC_COUNTER } from './constant.js';

​
​ // 数据定义
​ const initialState = {
​ counter: 0,
​ }

​
​ export const reduceCounter = function (state = initialState,action){
​ switch(action.type){
​ case ADD_COUNTER :
​ return {…state, counter: state.counter + action.num};
​ case DEC_COUNTER :
​ return {…state, counter: state.counter - action.num};
​
default :
return state;
}
}

index.js:

import { createStore, applyMiddleware, compose} from “redux”;
import { reduceCounter } from ‘./reducers.js’;

import thunkMiddleWare from ‘redux-thunk’;
// 这个函数是使用多个中间件，可以使用多个中间件
const storeEnhancer = applyMiddleware(thunkMiddleWare);

// 设置redux-devTools

const composeEnhancer = window.REDUX_DEVTOOLS_EXTENSION_COMPOSE || compose;

const store = createStore(reduceCounter,composeEnhancer(storeEnhancer));

export default store;
