vuex的模块化使用

vuex

vuex的作用：

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。通俗来讲就是管理多个组件的共享数据，如果多个组件共享数据使用事件总线会很麻烦，这种数据共享方式比较麻烦，所以有了一个统一管理共享数据的工具。

vuex的使用：

1.下载vuex插件：npm install vuex --save

2.在项目src目录下新建store文件夹，并且在下面新建一个index.js文件，如果其他文件导入该store文件，该文件文件下只有这个index文件，则默认会找到这个index文件。

3.index文件导入所需文件

import Vue from 'vue'

import Vuex from 'vuex'

Vue.use(Vuex);

4.编写vuex管理数据

`export default new Vuex.Store({

state:{ //此处存放需要管理的数据

​ token:”sdfsfsdfsdf”

},

mutations:{//此处存放更改数据的方法

changeToken:(state,value)=>{//方法需要传递两个参数，一个是仓库中的state，另一个是需要改变值

​ state.token = value

​ }

},

getters:{

​ getToken:(state)=>{ //getters用来取值，其效果和vue中的computed计算属性一样，可以用来包装数据和过滤数据

​ return state.token+”token令牌”

​ }

},

actions:{ //此处存放异步执行改变mutation中的方法

​ changeTokenasyn: ({commit},value)=>{

​ return commit(‘changeToken’,value)

​ }//或者

​ changeTokenasyn: (context,value)=>{

​ return context.commit(‘changeToken’,value)

}

}

})`

5.在main.js中引入store

import store from '../store'

在new Vue中添加store即可

6.在页面中使用vuex中的函数

1.触发getters的函数: this.$store.getters.gettersfunction(value1,value2)//可以传递参数

2.触发mutations的函数：this.$store.commit(‘vuexmutationsfunctionName’,value) / /value一般为一个参数或者将参数存放为对象

3.触发actions的函数：this.$store.dispatch(‘vuexactionsfunctionName’,value) //value和mutations一样

语法糖：

import { mapState, mapGetters, mapMutations,mapActions } from 'vuex'

computed:{

​ 1、…mapState([‘name’,’age’,’gender’]) //字符串中的的属性都是state中的属性，这样就可以在页面中直接用花括号使用，就是一个计算属性了。

​ 2、…mapState({

​ name:(state)=>state.name ,

​ storeName: (state)=>state.name, //当与数据仓库或者其他计算属性名字冲突时，可以使用该方法

​ // 传入字符串 ‘count’ 等同于 state => state.name

​ storeName: ‘name’,

​ }),

​ 3、computed:mapState({如同2的写法})

​ 1、…mapGetters({get:’getName’})

​ 2、…mapGetters([‘getname’,’getAge’])

​ mutations和actions都可以使用如下方式：

​ …mapMutations([‘changeName’,’changeAge’])

​ …mapActions([‘changeNameAsync’,’changeAgeAsync’])

}

模块化开发，降低耦合

在store文件夹下只留一个index文件用于整合其他模块，然后新建文件夹，并且为每个模块新建一个js文件，这个文件就是这个模块的vuex.

js模块文件结构：

const state = {}

const getters = {}

const mutations = {}

const actions = {}

export default {

state,

getters,

mutations,

actions

}

再在store下的index文件中引入该文件，并且在index文件中的

new Vuex.Store({

此处也可以添加state,getters,mutations,actions，模块化中，其中只有state的数据时分开的，其他的都是全局的在this.$store.getters||commit()||dispatch(),或者语法糖方法都可以访问到，而state数据分为rooState,模块state,在模块下可以state只能访问本模块下的state，也可以通过rootState进行根和兄弟之间的state的数据访问rootState.count //访问state的count，rootState.modulesA.name//访问兄弟的state

modules:{

person //在此添加进入引入的模块

}

})
