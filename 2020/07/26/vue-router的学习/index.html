vue-router的学习

vue-router的学习与使用：

1.下载vue-router(可以在创建项目的时候选择下载，此案例是创建后添加)

npm install vue-router --save

2.在src目录下创建router文件夹，并创建index.js文件，取名为index在别的文件引入中只需要写道router的目录，自动会寻找index.js文件，否则导入时需要指定文件名。

3.在index.js文件中：

import Vue from 'vue'

import VueRouter from 'vue-router'

Vue.use(VueRouter); //使用vue-router插件

const routers = [ //定义路由

{

path:'/home',

component:() => import ('../...'), //使用懒加载的方法能减少第一次加载的压力，按需加载

name: 'home',

meta:{ //元数据可以在$router.meta，$route.meta 中可以取到

tittle:''

}

}

]

const router = new VueRouter({

routers, //上诉的路由配置

mode:'history', //默认是hash模式，可以改成history模式，就没有#

linkActiveClass: 'active_class', //设置点所在的路由链接的样式名

})

router.beforeEach(from,to,next){ //设置全局路由守卫

next();

}

export default router; //5.导出router

4.在main.js文件中导入并使用router

import router from '../router'

注意：匹配的优先级，同一个路径可以匹配多个路由，此时，匹配的优先级就按照路由的定义顺序：谁先定义的，谁的优先级就最高。

router的构建选项

const router = new VueRouter({

routers:routers,

mode:’hash’（默认的hash）,’history’（设置为history需要后端配合配置）,’abstrac’(node环境没有浏览器环境)

linkActiveClass: string(激活样式的类名，然后可以再重新定义类，默认为router-link-active)

linkExactActiveClass：string 默认值: "router-link-exact-active"全局配置 `` 默认的精确激活的 class

})

Router-Link

router-link 的作用类似于a标签，但是他不会跳转页面刷新，其本质也是转化为a标签，

属性配置：

to属性

to=”/home/news” //to属性配置前端路由，需要配置完整的前端路由

也可以设置动态的前端路由，:to=”data”,使用v-bind语法绑定路由是可变的

tag属性：

tag=”button” 将该标签转化为button标签，默认是a标签

replace属性：

当标签中使用replace就会之前的路径不能返回。

active-class: 值： string

设定默认值: "router-link-active"

设置链接激活时使用的 CSS 类名，可以自己定义激活的样式。

router-view

name:

命名视图使用，当一个组件中含有多个router-view需要定义一个name来定义视图的渲染位置，配合router中配置的components属性，如果只有一个router-view的话，默认添加name属性值为default

编程式导航

$router.push

（跳转到某个页面）

this.$router.push('/home');

this.$router.push({path:'/home',query:{username:this.username}});

this.$router.push({name:'Home',params:{username:this.username}});

无论是使用申明式导航还是编程式导航，使用post方式传递参数时，都是利用命名路由的name属性来传递数据，如果使用了path属性则会忽略params属性，并且query（get方式，参数拼接在地址后面）在刷新后不会丢失数据，而params（post方式)在刷新后会丢失数据。

$router.replace

跟 router.push 很像，唯一的不同就是，它不会向 history 添加新记录，而是跟它的方法名一样 —— 替换掉当前的 history 记录。

$router.go

这个方法的参数是一个整数，意思是在 history 记录中向前或者后退多少步，类似 window.history.go(n)。

$router.back

$router.forward

$route对象

包含属性：

  • $route.path

    • 类型: string

      字符串，对应当前路由的路径，总是解析为绝对路径，如 "/foo/bar"。

  • $route.params

    • 类型: Object

      一个 key/value 对象，包含了动态片段和全匹配片段，如果没有路由参数，就是一个空对象。

  • $route.query

    • 类型: Object

      一个 key/value 对象，表示 URL 查询参数。例如，对于路径 /foo?user=1，则有 $route.query.user == 1，如果没有查询参数，则是个空对象。

  • $route.hash

    • 类型: string

      当前路由的 hash 值 (带 #) ，如果没有 hash 值，则为空字符串。

  • $route.fullPath

    • 类型: string

      完成解析后的 URL，包含查询参数和 hash 的完整路径。

  • $route.matched

    • 类型: Array

    一个数组，包含当前路由的所有嵌套路径片段的路由记录 。路由记录就是 routes 配置数组中的对象副本 (还有在 children 数组)。
