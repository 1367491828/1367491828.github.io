react 中setState的使用

setState在某些情况是异步，某些情况下是同步的.

异步的好处：

1.可以显著的提升性能 如果每次调用setState都进行一次更新，那么意味着render函数会被频繁调用，界面重新渲染，这样效率很低 最好的办法应该是获取到多个更新,之后进行批量更新

2.如果同步更新了state,但是还没有执行render函数,那么state和props不能保持同步 state和props不能保持一致性，会在开发中产生很多的问题。

所以会产生以下情况：

1

state = {
counter:0
}

changecounter = ()=>{ // 每点击一下会触发一下这个事件

​ this.setState({counter:this.state.counter + 10})

​ console.log(this.state.counter)

}

// 结果：第一次点击：0

// 第二次点击：1

// 第三次点击：2

因为是异步的，所以每次都是上一次的结果，并没有一下就更新state的值。

2

更会产生下面的问题：

函数执行下面的过程时：

this.setState({counter:this.state.counter + 1})

this.setState({counter:this.state.counter + 2})

console.log(this.state.counter)

// 结果：第一次点击：0

// 第二次点击：2

// 第三次点击：4

//上面产生的原因是因为操作的合并，最后的结果就是类似只执行了最后一次，因为每次拿到的都是旧的值。

3.

执行下面的操作：

this.setState({counter:this.state.counter + 10})
this.setState((state)=>({counter:state.counter + 1}))
this.setState((state)=>({counter:state.counter + 3}),()=>{
console.log(this.state.counter)
})

// 结果：第一次点击：14

// 第二次点击：28

// 第三次点击：42

this.setState((state)=>({counter:state.counter + 1})) 这种方法执行同步任务（相对的不是真的同步执行）

// 上面的结果是累加的

4.

this.setState({counter:this.state.counter + 10})
this.setState({counter:this.state.counter + 1})
this.setState((state)=>({counter:state.counter + 3}),()=>{
console.log(this.state.counter)
})

// 结果：第一次点击：4

// 第二次点击：8

// 第三次点击：12

5.

this.setState({counter:this.state.counter + 10})
this.setState((state)=>({counter:state.counter + 1}))
this.setState((state)=>({counter:state.counter + 3}),()=>{
console.log(this.state.counter)
})
// this.setState({counter:this.state.counter + 10})
this.setState((state)=>({counter:state.counter + 1}))
this.setState({counter:this.state.counter + 10},()=>{
console.log(this.state.counter)
})

// 结果：第一次点击：10

// 第二次点击：20

// 第三次点击：30

结论：如果最后一次是异步更新的方式，就只会执行最后一步的异步执行结果，如果最后一步是同步更新的方式，则结果为最后一步到倒数的一步结果相加（没有异步的就是全部同步的相加）。

redux学习

redux的使用：

1.下载redux:

npm install redux --save

2.redux的组成：（ action, reducer, store）

action: 是把数据从应用传到 store 的有效载荷。(就是设置传递（修改）数据固定格式，修改数据只能按照这个格式修改)。

reducer: 指定了应用状态的变化如何响应 actions,并发送到 store 的，记住 actions 只是描述了有事情发生了这一事实，并没有描述应用如何更新 state。

store:

  • 维持应用的 state；

  • 提供 getState() 方法获取 state；

  • 提供 dispatch(action) 方法更新 state；

  • 通过 subscribe(listener) 注册监听器;

  • 通过 subscribe(listener) 返回的函数注销监听器。

三大原则：

单一数据源。

整个应用的 state 被储存在一棵 object tree 中，并且这个 object tree 只存在于唯一一个 store 中。

State 是只读的。

唯一改变 state 的方法就是触发 action，action 是一个用于描述已发生事件的普通对象。

使用纯函数来执行修改（reducer必须是纯函数）。

为了描述 action 如何改变 state tree ，你需要编写 reducers。

Ajax的封装

function Ajax(options) {
return new Promise((resolve, reject) => {
let defaultOptions = {
method : ‘GET’,
contentType:’application/x-www-form-urlencoded’,
}
options = Object.assign(defaultOptions,options)
let { method, url, data, contentType } = options;
method = method.toUpperCase()
let xhr = window.XMLHttpRequest?new XMLHttpRequest():new ActiveXObject(‘Mscrosoft.XMLHttp’);
let str = ‘’;
for (let item in data) {
str = str + item + ‘=’ + data[item] + ‘&’
}
let len = str.length;
str = str.slice(0, len - 1);
if (method === ‘GET’) {
url = url + ‘?’ + str
}
xhr.open(method, url, true);
xhr.onreadystatechange = function(readystate) {
if (xhr.readyState === 4) {
if (xhr.status === 200 || xhr.status === 304) {
resolve(JSON.parse(xhr.responseText))
} else {
reject(xhr.status)
}
}
}
if (method === ‘POST’) {
xhr.setRequestHeader(‘Content-Type’, contentType)
if(contentType === ‘application/json’){
xhr.send(JSON.stringify(data))
}else if(contentType === ‘application/x-www-form-urlencoded’){
xhr.send(str)
}
} else {
xhr.send(null)
}
})
}

vue中错误合集

插值表达式的使用：

当插值表达式内的值或表达式返回的是 undefined,或 null插值表达式的内容为空，如果，花括号无值，则可能数据就是这两种情况(排除 ‘ ’ 空字符串)；

v-if的使用：

v-if为false的情况为： undefined， null, false, 0 , NaN, .

v-if 和v-for最好不要用在同一个元素上：解决办法如下：

1.为了过滤一个列表中的项目 (比如 v-for="user in users" v-if="user.isActive")。在这种情形下，请将 users 替换为一个计算属性 (比如 activeUsers)，让其返回过滤后的列表。（使用计算属性先将数据进行过滤，返回过滤后的可以渲染的结果进行渲染，这样就没有v-if）;

2.为了避免渲染本应该被隐藏的列表 (比如 v-for="user in users" v-if="shouldShowUsers")。这种情形下，请将 v-if 移动至容器元素上 (比如 ul、ol)。

v-for的使用

v-for 可以使用数组，数字，字符串，对象。

使用方法：v-for=”(item,idx,y) in num” ——

其中数字和数组，字符串只有两个参数，第一参数是值，第二个参数是对应的下标。

对象有三个值：第一参数是值，第二个参数是键，第三个是类似于下标的值。

v-on指令事件绑定的使用：

在v-on:click=’functionName()’简写为@click=’functionName()’

1.当方法不添加括号时，会默认传递一个参数，这个参数就是原生event对象。当有括号时，不添加参数时，就没有参数。

2.在函数传递this时，这个this的值为window对象。

3.需要传递event对象时，参数需要传递$event，不是原生的event，否则传递的是一个undefined，会报错。

4.当同时给一个事件绑定多个方法时，通过使用多个@click时只能触发第一个方法，第二个是无效的。

5.通过字符串给事件绑定多个方法，方法之间通过逗号隔开，@click=”fun1(),fun2()”，必须添加括号，否则无效。

6.通过数组绑定多个方法，@click=”[fun1($event),fun2()]”，这种方法也必须使用(),否则无效，建议使用这种方法给事件绑定方法。

7.通过对象方式，给不同事件绑定方法，v-on=”{click:fun1,mouseup:fun2}”这种方法只能使用v-on，不能使用@简写形式。这种不能给方法添加（）,否则会报错。也不能传递参数，只有一个默认的event对象作为默认参数。

8.通过v-on=”{click:[fun1,fun2]，mouseup:[fun1,fun2]}”这种方式可以给相同的事件，绑定多个方法,这种方法也是不能传参的，默认只有$event参数，加括号会报错。

总结以上只有在给事件绑定多个事件多个方法以数组形式或对象方式才不能添加括号，且不能传参，只有一个默认的event对象。不加括号还主要使用在父子组件之间的子给父传递参数中常见。

子组件给父组件传递数据也通过自定义事件来传递数据，但是这种方式是通过触发事件来控制数据的传递，不具有数据响应式。

Vue的仓库属性

vue实例中的data属性：

1.在组件中data必须是一个函数，返回一个对象，原因：因为组件可能被多次使用，如果被用来创建多个实例，如果 data 仍然是一个纯粹的对象，则所有的实例将共享引用同一个数据对象！通过提供 data 函数，每次创建一个新实例后，我们能够调用 data 函数，从而返回初始数据的一个全新副本数据对象。简而言之，就是 data 中数据可能会被复用，要保证不同组件调用的时候数据是相同的。

2.在data数据仓库中属性不能以_下划线或者$符开头，否则会报错。

3.在未拿到请求数据时，如果数据中有对象嵌套，未拿到数据，页面渲染了，里面的是不存在的对象的属性取值时，会报错。例如数据仓库中有info:{},页面取值：,后台拿到数据后赋值给info,首先在页面没有拿到数据时会报错，并且页面不会渲染出来，在拿到数据后会正常渲染，控制台会留下报错。

解决办法1：在这个组件根组件上添加v-if=”loading”,loading先取值为false,v-if的原理是，如果为false就不进行渲染，而v-show是添加style=”display:none”,所以只要不先渲染，就不会报错，v-show不能在这里使用，只能v-if.

解决办法2：如果嵌套的数据对象不深，可以先进行预定义。比如info:{person:{}},页面使用,这种情况虽然不会报错，但是可能会显示空白，取到数据后再出现。

上面两种情况都是在网速条件不好的时候对用户体验是极差的。在上面的情况可以使用loading状态。

4.如果想给属性中再添加属性，这样直接添加属性的话，并不会产生响应式，因为对象添加属性，响应式原理并不能监控到，要使用this.$set(obj,’key’,value);这个方法不能给实例添加属性，也不能给数据仓库data添加属性。所以在组件定义时就需要把需要的数据先定义在数据仓库中。也可以通过变异方式实现this.obj = {…this.obj,key:value},这种方法可以给对象重新赋值，这种做法通过给对象重新赋值，vue会重新给对象进行数据拦截，产生响应式。

5.如果对数据仓库中的数组进行变动要实现响应式，只能通过其变异方法，push,pop,shift,unshift,reverse,splice,sort变异方法，他们都是能够实现响应式的，比如修改数组下标为1的值，使用this.arr[0] = 10,这种方法是不能实现响应式效果的，vue工具中是看不到的，如果把vue实例赋值给变量vm

使用vm.$data可以看到数据被修改了，vue的工具不能捕获到，可以使用this.$set(arr,’1’,10) 或者 this.splice(1,1,10),两种方式都能实现响应式，直接修改数组的值或者数组长度都不能产生响应式效果，只能使用变异方法变通实现。

vue-router的使用

动态路由的使用：

在rouetr/index.js中路由配置为{ path:’/:id’ , component: Component}

这种带有：的路由就是动态路由，可以通过此方式来动态获取参数，且不会丢失参数，

获取参数方法：this.$route.params.id

路由的匹配方式，先匹配到谁就显示谁：

{

​ path:’/test’,

​ component:Test

​ path:’/:id’,

​ component:HelloWorld

这种方式如果匹配到/test,就是test组件，如果没匹配到就是下一个/：id，所以如果动态路由配置在前面，当只有一个路由/xxx的时候就都会被动态路由拦截。不会到test路由，比动态路由更多级将不会适用动态路由。

响应路由参数的变化

path 和 url 的区别

使用react-router-dom的useRouteMatch()钩子函数（函数组件中才能使用）

在 match 属性中，有 path 和 url 属性，大部分时间他们是一样的，那么区别是什么呢？

假如路由匹配路径是 /Params/2/:myParams，实际跳转路径是 /Params/2/1（此时匹配成功）。

那么；

  1. url：/Params/2/1
  2. path：/Params/2/:myParams
  3. 所以URL一般用于Link标签中
  4. path一般用于Route标签中
let { path, url } = useRouteMatch();
